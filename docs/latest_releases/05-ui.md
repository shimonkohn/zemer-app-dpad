# 05 — The UI

Two surfaces render the feed, both observing `LatestReleasesViewModel.releases`: the Home shelf
and the "See all" screen. Tap behaviour is shared between them via one helper
(`LatestReleasePlayback.openOrPlay`), and both reuse the `subtitleOverride` card param to show
`Artist • <relative date>`.

## The Home shelf (`ui/screens/HomeScreen.kt`)

### Wiring

- The feed ViewModel is obtained independently of `HomeViewModel`
  (`:140-141`):
  ```kotlin
  val latestReleasesViewModel: LatestReleasesViewModel = hiltViewModel()
  val latestReleases by latestReleasesViewModel.releases.collectAsState()
  ```
- The shelf is **capped at 12** items (`:166`):
  ```kotlin
  val latestReleasesCapped = remember(latestReleases) { latestReleases.take(12) }
  ```
- A non-empty feed counts as remote Home content for the shimmer/empty logic
  (`hasRemoteHomeContent`, `:433-438`), so the feed alone is enough to show Home as populated.

### Placement and render (`:529-578`)

The shelf sits **directly above Featured Playlists** (`featured_playlists_title` follows at
`:581`). It is two lazy items — a header and the list:

```kotlin
latestReleasesCapped.takeIf { it.isNotEmpty() }?.let { releases ->
    item(key = "latest_releases_title", contentType = "header") {
        NavigationTitle(
            title = stringResource(R.string.latest_releases),
            onClick = { navController.navigate("latest_releases") },   // "See all"
            modifier = Modifier.animateItem()
        )
    }
    item(key = "latest_releases_list", contentType = "grid") {
        LazyRow(
            contentPadding = WindowInsets.systemBars.only(WindowInsetsSides.Horizontal).asPaddingValues(),
            modifier = Modifier.animateItem()
        ) {
            items(items = releases, key = { it.browseId }, contentType = { "album" }) { release ->
                val album = remember(release.browseId) { release.toAlbumItem() }
                val dateLabel = remember(release.browseId) { release.relativeDateLabel() }
                YouTubeGridItem(
                    item = album,
                    subtitleOverride = joinByBullet(release.artistName, dateLabel),   // "Artist • 3 days ago"
                    centeredPlayButton = release.isPlayableSingle(),
                    isActive = release.isNowPlaying(mediaMetadata),
                    isPlaying = isPlaying,
                    coroutineScope = scope,
                    thumbnailRatio = 1f,
                    modifier = Modifier.combinedClickable(
                        onClick = { release.openOrPlay(navController, playerConnection, database) },
                        onLongClick = { /* haptic + YouTubeAlbumMenu */ }
                    )
                )
            }
        }
    }
}
```

Key points:

- **Header tap = "See all":** the `NavigationTitle.onClick` navigates to the `latest_releases`
  route. (Featured Playlists' header has no `onClick` — only Latest Releases offers See-all.)
- **Visibility:** `takeIf { it.isNotEmpty() }` — when the filtered feed is empty, the header and
  list are not emitted at all.
- **Layout:** `LazyRow` of `YouTubeGridItem` square cards (`thumbnailRatio = 1f`), matching the
  Featured Albums/Artists/Videos shelves. This is the modern shelf idiom the rest of Home uses.
- **Active state:** `isActive = release.isNowPlaying(mediaMetadata)`
  (`latestreleases/LatestReleasePlayback.kt`). A single plays as a videoId with no album bound, so
  it matches on the **track id** (`mediaMetadata?.id == sampleVideoId`); an album is active when a
  track from it is playing (`mediaMetadata?.album?.id == browseId`). When active, the centred play
  overlay clears and the card shows its playing state — a plain `album?.id` check would never match
  a playing single.
- **Subtitle:** `subtitleOverride = joinByBullet(release.artistName, dateLabel)` shows
  `Artist • <relative date>` where the card's subtitle normally goes. `joinByBullet` drops
  null/empty parts, so an unparseable date degrades to just the artist (never the default
  `AlbumItem` subtitle).
- **Tap behaviour:** `release.openOrPlay(...)` — a single (`trackCount == 1`) plays immediately
  with autoplay radio, anything else opens `album/<id>` (see "Tap behaviour" below). Long-press ->
  `YouTubeAlbumMenu` (standard album card behaviour).

## The "See all" screen (`ui/screens/LatestReleasesScreen.kt`)

A `@Composable fun LatestReleasesScreen(navController, scrollBehavior, viewModel = hiltViewModel())`
(`:47-51`) showing the **full** feed (no `take(12)`) as a vertical list:

- A `LazyColumn` padded with `LocalPlayerAwareWindowInsets` (`:60-62`).
- Each row is a `YouTubeListItem` (the row form, appropriate for a full-screen list) with the
  same `subtitleOverride = joinByBullet(artistName, dateLabel)`, the same
  `centeredPlayButton = release.isPlayableSingle()` (a single shows the centred play button on its
  row thumbnail; albums show none), and the same `release.openOrPlay(...)` tap and
  `YouTubeAlbumMenu` long-press as the Home shelf — the behaviour is shared, not re-implemented.
- A standard `TopAppBar` titled `R.string.latest_releases`, back button using the app's
  `IconButton` with `onClick = navigateUp` / `onLongClick = backToMain` (`:92-106`) — exactly the
  pattern `docs/ui/standards.md` section 2 prescribes.
- "When the feed is empty/unavailable the list is simply empty — nothing is forced" (`:41-43`).

### Route (`ui/screens/NavigationBuilder.kt:87-88`)

```kotlin
composable("latest_releases") {
    LatestReleasesScreen(navController, scrollBehavior)
}
```

Both `hiltViewModel()` call sites resolve to a `LatestReleasesViewModel`; the underlying
`LatestReleasesStore` is a process-wide `object`, so the cached feed and once-per-launch refresh
are shared across the Home shelf and the See-all screen.

## Tap behaviour — `openOrPlay` (`latestreleases/LatestReleasePlayback.kt`)

Both surfaces route a card tap through one shared extension so the behaviour can't drift:

```kotlin
fun LatestRelease.openOrPlay(navController, playerConnection, database) {
    val single = playableSingle()                                   // non-null only for a 1-track single
    if (single != null) playerConnection.playQueue(YouTubeQueue.radio(single, database))
    else navController.navigate("album/$browseId")
}
```

- `playableSingle()` (`:22`) is the **pure, Android-free** core of the decision: it returns the
  track's `MediaMetadata` only when `trackCount == 1` and a `sampleVideoId` exists, and `null`
  otherwise (a real album, or an older cached feed with no track count). It is unit-tested without
  a player or Android runtime (doc 06).
- A **single** plays immediately via `YouTubeQueue.radio` — the same autoplay-radio queue the rest
  of Home uses (`:42`), so music continues after the track.
- Anything else **opens the album page** (`:44`) — the prior behaviour, and the safe fallback
  whenever `trackCount` is absent.
- Long-press is unchanged on both surfaces: `YouTubeAlbumMenu`.

**Visual affordance.** A release the tap will actually play (`isPlayableSingle()` — `trackCount == 1`
with a videoId) shows the song-style **centred** play button on its artwork (the same
`OverlayPlayButton` the Keep Listening song cards use), on both the Home card and the See-all row.
An album keeps the **corner** `AlbumPlayButton` on the grid card and no button on the list row.
Both surfaces pass this via the `centeredPlayButton` flag, so the icon never promises playback the
tap won't deliver (the flag and `openOrPlay` read the same predicate — pinned by the test in doc 06).

## Card params: `subtitleOverride` + `centeredPlayButton` (`ui/component/Items.kt`)

Both grid- and list-item composables gained two optional params, used identically so the two
surfaces stay symmetric (no new component, no duplicated card — consistent with
`docs/ui/standards.md` section 1, reuse before building):

- **`subtitleOverride: String? = null`** — replaces the computed subtitle (here, to put
  `Artist • <relative date>` on the card). When null the card computes its normal subtitle (for an
  `AlbumItem`: `artists` and `year`). `YouTubeListItem` param `:839`, used at `:871`;
  `YouTubeGridItem` param `:943`, used at `:958`.
- **`centeredPlayButton: Boolean = false`** — when true the artwork shows the centred
  `OverlayPlayButton` (the song-card style) instead of an album's corner `AlbumPlayButton` (grid) /
  no button (list). `YouTubeGridItem` swaps the button on the flag (`:989-996`); `YouTubeListItem`
  overlays it on the row thumbnail (`:879-893`). Default false, so every other caller is unchanged.
